# Security Policy

## Supported versions

The project is pre-alpha. Security fixes are applied only to the latest version on the default branch.

## Reporting a vulnerability

Use GitHub's private security advisory feature for this repository. Do not open a public issue for a vulnerability that could expose credentials, private data, deployment systems, or users.

Include:

- the affected component and version;
- reproduction steps or a proof of concept;
- expected and observed behavior;
- potential impact;
- any suggested mitigation.

Maintainers should acknowledge a report within seven days. Resolution timing depends on severity and maintainer availability.

## Data safety

The project must not accept student-level personal information, private family profiles, credentials, or unpublished education records. If such data are committed or posted, stop redistribution, notify maintainers privately, and follow a documented removal and credential-rotation process.
